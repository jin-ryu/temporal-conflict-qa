# 6. 실험 설계

## 6.1 연구 질문 (Research Questions)

**RQ1 (Emergence):** SFT에서 출력 형식만 학습시킨 후 GRPO로 강화할 때, `<reasoning>` 블록에 temporal reasoning이 자발적으로 emerge하는가?

**RQ2 (효과성):** TV-RAG는 기존의 파이프라인 조작 방식(리랭킹, CoT 프롬프팅) 및 자가 검증 방식(Self-RAG) 대비 시간적 충돌 해소에서 우수한 성능을 달성하는가?

**RQ3 (구성요소 기여):** 보상 함수의 곱셈 구조($R_{answer} \times R_{evidence}$), `<reasoning>` 블록, teacher reasoning cold-start 각각이 성능에 어떻게 기여하는가?

## 6.2 Pilot Study — 기존 LLM의 시간적 충돌 해소 능력 평가

본 실험에 앞서, 기존 LLM이 시간적 충돌 상황에서 어떤 수준의 성능을 보이는지 파일럿 스터디를 수행한다. TC-RAG Dataset의 일부를 사용하여 모델 규모와 유형에 따른 성능 변화를 관찰한다.

**평가 대상 모델:**

| 모델 | 크기 | 유형 | 목적 |
|------|------|------|------|
| GPT-4.1 | — | 상용 API | 상용 대형 모델 — 규모와 상관없이 temporal conflict에 취약한지 검증 |
| Meta-Llama-3.1-70B-Instruct-AWQ-INT4 | 70B | 오픈 (AWQ INT4) | 오픈 모델 대표 — 대형 오픈 모델도 temporal conflict에 취약한지 검증 |

- 상용 API(GPT-4.1)는 비용 제약으로 평가 규모를 제한하며, 오픈 모델은 H100 80GB에서 vLLM으로 실행하여 **500건 이상**으로 확대한다

**실험 조건:**

| 조건 | 키워드 | 시간 충돌 | 질문 시간 힌트 | 대상 mode |
|------|--------|----------|--------------|-----------|
| **충돌 없음** | `no_conflict` | 없음 (outdated 청크 제거) | 있음 | current |
| **충돌 있음** | `conflict` | 있음 (전체 청크) | 있음 | current, outdated_* |
| **충돌 + 모호** | `ambiguous` | 있음 (전체 청크) | 없음 | current_raw |

**조건 설계 의도:**

- **no_conflict → conflict**: 충돌 정보(outdated 청크) 존재 여부가 성능에 미치는 영향
- **conflict → ambiguous**: 질문의 시간 힌트 유무가 성능에 미치는 영향
- **no_conflict vs ambiguous**: 충돌 + 모호성의 복합 효과

**가설:**

- **H1**: no_conflict → conflict 성능 하락 (충돌 정보가 혼란 유발)
- **H2**: conflict → ambiguous 추가 하락 (시간 힌트 없으면 더 어려움)
- **H3**: evidence accuracy가 answer accuracy보다 더 크게 하락
- **H4**: 상용 대형 모델(GPT-4.1)과 오픈 70B 모델 모두 temporal conflict 조건에서 일관된 성능 하락

이 파일럿 결과는 "기존 LLM은 모델 유형과 규모에 관계없이 시간적 충돌에 취약하다"는 문제 제기의 실증 근거로 활용한다.

## 6.3 데이터셋

### 6.3.1 데이터셋 총괄

| 데이터셋 | 역할 | 규모 (목표) | 생성 모델 | 용도 |
|----------|------|------------|-----------|------|
| **TC-RAG Dataset** | 학습 + In-domain 평가 | HoH 전체 96K건 기반, Train 80% / Test 20% | Meta-Llama-3.1-70B-Instruct-AWQ-INT4 (vLLM) | SFT, GRPO, 메인 평가, Ablation |
| **HoH 원본** | Near-domain 평가 | 논문 보고 수치 활용 + TV-RAG 직접 평가 | — | 과적합 검증, 기존 LLM 대비 비교 |
| **SituatedQA** | Out-of-domain 평가 | 수백 건 subset 구성 | — | Zero-shot 전이, 일반화 검증 |

### 6.3.2 TC-RAG Dataset (학습 + In-domain 평가)

§4.2.1에서 구축한 TC-RAG Dataset을 사용한다. 각 샘플은 `hoh_source_idx` 필드(원본 HoH 인덱스)를 가지며, 이를 기준으로 Train/Test를 분할하여 데이터 누수를 방지한다. 데이터 생성에는 Meta-Llama-3.1-70B-Instruct-AWQ-INT4를 사용하며, H100 80GB 단일 GPU에서 vLLM으로 서빙한다.

| mode | 설명 | 시간 힌트 | 용도 |
|------|------|-----------|------|
| `current` | LLM 생성 질문, 최신 답이 정답 | 있음 | SFT, GRPO, 평가 |
| `outdated_i` | LLM 생성 질문, 과거 답이 정답 | 있음 | GRPO, 평가 |
| `current_raw` | HoH 원본 질문 그대로 (LLM 호출 불요) | 없음 | GRPO, 평가 |

### 6.3.3 HoH 원본 벤치마크 (Near-domain 평가)

**HoH 논문의 실험 설계.** Ouyang et al. (2025)은 Wikipedia 월별 스냅샷(2024.06–11)을 토큰 수준 diff로 추적하여 96K건의 QA를 구축하고, Relevant(R)·Outdated(O)·Distracting(D) 패시지의 조합을 제어하여 실험했다. 평가 지표는 LLM-as-judge(Llama-3.1-70B) 기반으로 Perfect(+1) / Missing(0) / Harmful(−1)을 판정하며, Score = Perfect% − Harmful% (범위 −100 ~ +100)를 사용한다. 예산 제약으로 상용 모델은 테스트하지 않았으며, 오픈소스 모델 3종(Llama-3.1-70B-Instruct, Llama-3.1-8B-Instruct, Qwen-2.5-7B-Instruct)을 평가했다. 핵심 발견은 현재 정답이 컨텍스트에 포함되어 있어도 outdated 패시지가 단 1개만 공존하면 Llama-70B 기준 Score가 89 → 65로 약 24점 하락하며, Distractor 6개 추가 시 2% 미만 변동과 대조적이라는 것이다. MRAG·TempRALM 등 시간 인식 검색 기법을 적용해도 생성기의 교란은 방어되지 않았다.

**본 연구의 평가 계획.** TC-RAG가 HoH에서 파생된 데이터셋이므로, HoH 원본 평가 프로토콜을 적용하여 TV-RAG를 평가하고, 그 결과를 다음과 비교한다:

- **HoH 논문의 오픈소스 모델 결과와 비교**: HoH 논문에서 보고된 Llama-3.1-8B / Qwen-2.5-7B 결과와 비교하여, 동일 규모 모델 대비 RL 학습이 가져온 temporal conflict 해소 능력 향상 폭을 직접 측정
- **대형 모델 대비 위치 확인**: 파일럿 스터디(6.2)에서 확인된 대형 모델(GPT-4.1·Llama-3.1-70B)의 취약성을 기준으로, 7B TV-RAG가 이를 극복하여 대형 모델 수준 이상의 성능을 달성하는지 확인
- **과적합 검증**: TC-RAG 테스트셋에서의 성능이 HoH 원본에서도 유지되는지 비교한다. HoH 원본에서 성능이 크게 하락한다면 TC-RAG 형식에만 과적합된 것이고, 유사한 수준을 유지한다면 범용적인 temporal reasoning을 학습한 것으로 판단한다

### 6.3.4 SituatedQA (Out-of-domain 평가)

SituatedQA(Zhang & Choi, 2021)는 시점에 따라 정답이 달라지는 temporal QA 벤치마크다. GitHub(`mikejqzhang/SituatedQA`)에서 직접 다운로드한다.

**데이터 구조.** 데이터는 세 폴더로 구성되며, 평가에는 주로 `qa_data/`와 `raw_data/`를 사용한다.

**`qa_data/temp.test.jsonl`** — 질문-시점 쌍으로 펼친 메인 평가 데이터 (~520건):

```json
{
  "question": "who is president of india in present time",
  "edited_question": "who is president of india in present time as of December 29, 2020",
  "date": "December 29, 2020",
  "date_type": "sampled_date",
  "answer": ["Ram Nath Kovind"],
  "any_answer": ["Ram Nath Kovind", "Pranab Kumar Mukherjee"]
}
```

- `answer`: 해당 시점의 정답
- `any_answer`: 모든 시점에서 유효했던 정답의 합집합 — 모델의 오답이 시간 혼동(temporal confusion)인지 완전한 오류인지를 구분

**`raw_data/temp.test.jsonl`** — 질문별 전이 타임라인 원본 (고유 질문 ~100건):

```json
{
  "question": "who is president of india in present time",
  "is_dependent": true,
  "timelines": [
    {
      "prev_answers": ["Pranab Kumar Mukherjee"],
      "prev_start": {"year": 2012, "month": 7, "date": 25},
      "cur_answers": ["Ram Nath Kovind"],
      "cur_start": {"year": 2017, "month": 7, "date": 25}
    }
  ]
}
```

- `timelines`: 정답이 바뀌는 전이점과 이전/현재 정답을 구조화. 충돌 evidence passage 구성 시 이 전이 정보를 활용

**규모.** temporal 질문 기준 고유 질문 약 300건 (train/dev/test), 질문-시점 쌍으로 펼치면 약 2,600건 (`data/qa_data/temp.*.jsonl`).

**평가 방식.** `is_dependent: true`인 질문에 대해, `timelines`의 전이 정보를 활용하여 서로 다른 시점의 정답을 뒷받침하는 두 개의 evidence passage를 구성한다 (예: `cur_start` 이후 문서 vs `prev_start` 이후 문서). 이를 TV-RAG 입력 포맷으로 변환하여 시간적 충돌 상황을 재현하고, `edited_question`의 시점에 맞는 정답을 선택하는지 평가한다. `any_answer`를 활용하여 temporal confusion rate(시간 혼동으로 인한 오답 비율)도 별도 측정한다.

**목적.** TC-RAG는 narrative hint("shortly before the update" 등)만으로 학습하며, 명시적 연도 질문("as of 2024")은 학습에 포함하지 않는다. SituatedQA는 명시적 연도 형식의 질문으로 구성되어 있으므로, 이 평가를 통해 narrative hint로 학습한 temporal reasoning이 명시적 연도 질문에도 zero-shot으로 전이되는지를 검증한다. 전이에 성공하면 모델이 표면적 패턴이 아닌 범용적인 시간적 추론 능력을 학습했다는 강한 증거가 된다.

**성능 미달 시 대안.** zero-shot 전이 성능이 기대에 미치지 못할 경우, 다음 두 가지 접근을 future work으로 검토한다:

1. **명시적 연도 질문 생성**: 기존 `chunks_to_qa.py` 파이프라인에서 프롬프트를 변경하여 "as of {date}" 형식의 질문을 추가 생성하고 학습 데이터에 포함
2. **SituatedQA 추가 학습**: SituatedQA 데이터를 TV-RAG 입력 포맷으로 변환하여 GRPO 학습에 포함, 도메인 다양성을 확보

## 6.4 비교 모델 (Baselines)

### 6.4.1 파이프라인 조작 기반 (RQ2 검증)

**1. Standard RAG**

- 검색된 문서를 추가 처리 없이 베이스 모델(Qwen2.5-7B)에 제공하는 기본 제로샷 파이프라인
- 목적: 시간적 충돌 해결 능력의 하한선(lower bound) 측정

**2. Time-aware Reranking**

- 검색기의 유사도 점수에 타임스탬프 기반 지수 감쇠 함수를 곱하여 문서 순위를 재조정한 뒤 생성기에 제공
- 목적: 검색 파이프라인 측 시간적 개입의 한계 확인 (outdated mode 질문에서 최신 편향으로 실패)

**3. Time-aware CoT Prompting**

- Standard RAG에 "각 문서의 작성 날짜를 비교하고 시간적 선후 관계를 추론한 뒤 답변하라"는 CoT 지시 추가
- 목적: 프롬프트 엔지니어링 기반 외부 추론 지시 대비, RL 내재화의 효과 비교

### 6.4.2 자가 검증 기반 (RQ2 검증)

**4. Self-RAG (7B)**

- 검색 필요성 판단, 관련성·지지 여부·유용성을 자가 평가하는 reflection 토큰을 생성하도록 학습된 모델 (Asai et al., ICLR 2024 Oral)
- 사용 모델: `selfrag/selfrag_llama2_7b` (HuggingFace pretrained, inference만 수행)
- 목적: 의미적 관련성 기반 자가 검증이 시간적 충돌을 구분할 수 있는지 검증. Self-RAG의 reflection 토큰(`[IsRelevant]`, `[IsSupported]`)은 시간적 유효성을 평가하도록 설계되지 않았으므로, 구버전 문서가 의미적으로 관련 있으면 '유용한 정답'으로 오판할 것을 기대
- 참고: 결과 테이블에 Self-RAG 13B 수치도 함께 기재하여, TV-RAG 7B가 더 큰 모델과도 비등하거나 우수한지를 보인다

### 6.4.3 학습 기반 (RQ1/RQ3 검증)

**5. SFT-only**

- GRPO 학습 없이 format SFT만 적용한 모델. 베이스 모델의 사전학습 지식만으로 temporal conflict를 어느 정도 해소할 수 있는지를 측정
- 목적: GRPO의 순수 기여를 SFT 대비 분리 검증. SFT-only도 잘하면 GRPO 불필요, SFT-only는 못하고 GRPO 후 잘하면 RL이 temporal reasoning을 학습시킨 것

### 6.4.4 대형 모델 (RQ2 검증)

**6. Large LLM (Meta-Llama-3.1-70B-Instruct-AWQ-INT4 / GPT-4.1)**

- 70B급 오픈 모델 및 상용 대형 모델에 Standard RAG 방식으로 평가 (추가 학습 없음)
- 목적: 모델 규모만으로 시간적 충돌이 해소 가능한지 검증. "70B/상용 모델도 못 하는 걸 7B TV-RAG가 해냄" → RL 학습의 가치를 실증
- 파일럿 스터디(6.2) 결과를 그대로 활용

## 6.5 평가 지표

### 6.5.1 End-to-End 성능

| 지표 | 설명 |
|------|------|
| Answer EM | exact match (정규화 후 완전 일치) |
| Answer Token F1 | 토큰 단위 precision/recall F1 |
| Evidence Accuracy | `<evidence>` 블록의 chunk id가 ground truth evidence_chunk_id와 일치 |
| Combined Score | Answer Token F1 × Evidence Accuracy |

Combined Score는 EM이 아닌 **Token F1 기반**으로 산출한다. EM은 모델이 정답과 정확히 동일한 문자열을 출력해야만 1이 되므로, 정답 핵심 토큰을 포함하더라도 표현이 조금만 달라지면(예: "Ram Nath Kovind" vs "Ram Nath Kovind is the current president") 0이 된다. 이 경우 Evidence Accuracy가 1.0이어도 Combined Score가 0이 되어 모델의 temporal reasoning 능력을 과소평가하게 된다. Token F1은 부분 일치를 반영하므로 Combined Score의 변별력이 유지된다. EM은 별도 참고 지표로 함께 보고한다.

Combined Score의 곱셈 구조는 보상 함수의 $R_{answer} \times R_{evidence}$와 동일한 논리를 따른다. 올바른 문서를 선택하지 않으면 answer가 맞더라도 점수가 0이 되어, shortcut을 통한 정답 달성을 배제한다.

Evidence Accuracy는 `<evidence>` 태그 기반 구조화 출력을 사용하는 학습 기반 모델(Baseline 5, TV-RAG)에만 적용된다. 파이프라인 조작 기반(Baseline 1-3)과 대형 모델(Baseline 6)은 자유 생성 구조이므로 Answer EM / F1로만 비교한다. Self-RAG(Baseline 4)는 자체 reflection 토큰으로 문서를 선택하므로, 해당 선택을 Evidence Accuracy에 매핑하여 측정한다.

### 6.5.2 Temporal Reranking 효과 검증

Generator는 답변 생성과 별개로, `<evidence>` 태그로 문서를 선택하는 행위 자체가 Listwise Temporal Reranker 역할을 한다. 이 리랭킹 능력만을 독립적으로 평가하기 위해, §5절의 Plug-in 필터 모드를 실증한다.

| 설정 | 방식 | 측정 |
|------|------|------|
| Baseline | 랜덤 순서 청크 → 대형 LLM(Qwen2.5-72B 등)이 바로 답변 | Answer EM / F1 |
| Ours (Plug-in) | 랜덤 순서 청크 → TV-RAG가 `<evidence>`로 골드 문서 선택 → **해당 문서만** 대형 LLM에 전달 → 답변 | Answer EM / F1 + Evidence Accuracy |

TV-RAG의 Standalone 성능(6.5.1)과 Plug-in 필터 성능(6.5.2)을 함께 보고하여, 단독 사용과 리랭커 활용 두 시나리오 모두에서의 가치를 보인다.

**추가 분석 — Positional Bias**: evidence 청크가 context 내 몇 번째 위치에 있을 때 정답률이 어떻게 달라지는지를 분석한다. Baseline에서는 positional bias가 성능에 영향을 주지만, 본 모델의 `<evidence>` 선택이 위치와 무관하게 올바른 문서를 고르는지를 보여준다.

### 6.5.3 Emergence 검증 — `<reasoning>` 블록 분석

SFT가 format만 가르쳤으므로, GRPO 이후 `<reasoning>` 블록에 temporal reasoning이 나타나는지를 post-hoc으로 검증한다.

1. **LLM-as-judge**: `<reasoning>` 블록에 (a) temporal reasoning이 포함되어 있는가, (b) 논리적으로 유효한가를 분류
2. **조건별 비교**: 시간 힌트 있는 질문(current/outdated) vs 없는 질문(current_raw)에서 reasoning 패턴 차이 분석
3. **Ablation 대비**: SFT(format-only)+GRPO vs SFT(teacher reasoning)+GRPO에서 `<reasoning>` 품질 비교

---

## 6.6 Ablation 실험 (RQ3)

| Ablation | 목적 | 근거 |
|---|---|---|
| SFT(format-only) + GRPO vs SFT(teacher reasoning) + GRPO | teacher reasoning cold-start의 기여도 검증 (메인 실험은 format-only) | format-only에서도 reasoning이 emerge하면 teacher reasoning은 선택적 보조 수단 |
| `<reasoning>` 포함 vs 미포함 (`<evidence>` + `<answer>`만) | `<reasoning>` 블록 자체의 성능 기여 검증 | CoT 계열 연구(Wei et al., 2022; Self-RAG, 2023)는 수학/RAG에서 reasoning 출력이 성능을 올림을 보였으나, temporal conflict RAG에서는 미검증 |
| $R_{answer}$ only vs $R_{answer} \times R_{evidence}$ | evidence reward의 필요성 검증 | evidence reward 없이 answer만 맞히는 shortcut이 존재함을 보임 |
